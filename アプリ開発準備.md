# VS Code + GitHub + 仮想環境 完全ガイド
#GitHub #重要
## はじめに

このテキストは、

- VS Code
- GitHub
- Python仮想環境(.venv)

を組み合わせて、
安全かつ効率的にPython開発を行うための
永久保存版ガイドです。

特に、

- 複数PC運用
- GitHubをマスタ管理にする
- VS Code中心の開発
- pandasなど外部ライブラリ利用

を前提にしています。

---

# なぜ GitHub + 仮想環境 を使うのか？

## GitHubの役割

GitHubは単なるコード置き場ではありません。

本質は：

- 履歴管理
- バックアップ
- 複数PC同期
- 完成版管理

です。

例えば：

- 昨日の状態に戻す
- PC故障時に復元
- 自宅PC → 会社PCへ移行

などが可能になります。

---

## 仮想環境(.venv)の役割

Pythonでは、
プロジェクトごとに必要ライブラリが異なります。

例えば：

- Flask
- pandas
- pywebview
- OpenCV

など。

これらを混在させると環境破損しやすくなります。

そこで：

「プロジェクト専用Python環境」

を作ります。

これが `.venv` です。

```text

```

```text
履歴管理
```

Gitの方が開発向き。

# Google DriveでGit管理しない方がよい理由

特に：

```text
.git/
```

との相性が悪い。

同期競合で破損可能性あり。

推奨：

```text
D:\\GitHub
```

はGoogle Drive対象外。

# VS Codeが仮想環境を自動認識する理由

`.venv/`

という名前はVS Codeが優先検出。

そのため：

```text

```

と自動選択されやすい。
